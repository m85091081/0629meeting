基於 RTSP & WebRTC 的語音辨識雲化系統

傳統架構 雲分散架構
協定/運算比較
音訊傳輸協定 無需音訊傳出採用傳統客戶端辨識 採用WebRTC & RTSP 即時串流
辨識資料協定 寫死在 Client 端，且多數無開放更新協定 採用伺服器辨識，資料以 Web-Socket 進行傳輸
辨識單元 需基於嵌入裝置例如 Arduino 等 可為任何裝置/電腦/穿戴式
運算單元 由Client進行即時運算辨識且非同步 由Cloud Server 針對 WebRTC 所夾帶的串流訊息即時辨識
優劣分析
擴充性 劣,需受廠商限制綁定特殊裝置 好,採用通用的裝置以及基於Python的可程式化編程
開發難度 劣,需受限於特定的裝置(Arduino/或者封閉的晶片) 好,可基於任何裝置以通用的 WebRTC 協定以及 Python 串 Hadoop 進行開發
硬體成本 劣,需受限於特定的裝置(Arduino/或者封閉的晶片) 好,可基於所有目前的裝置進行開發

結論:以雲化開發符合我們的專題宗旨，並且可以作為我們專題的基底。
且可以透過此系統深度結合原本學長姐的系統，並且改善原始的盲點，
例如飛行器急救包系統，如果遇到飛行器死角就有可能失去一條人命，
假設透過雲化語音辨識系統結合飛行器系統，就可以順利的改善此問題。
且可以以低成本高擴充性，高底層技術，作為雲運算平台的最佳實作。
